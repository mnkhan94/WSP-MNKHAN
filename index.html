Drop-down menu Header

What I hope to practice/learn?

Even the most simple of headers can have its own issues when deciding to float or inline-block the navigation. The addition of drop-down menu items always seems to complicate things even more. This component was selected in order to practice this technique. The ability to explore creating an eCommerce cart feature is a bonus.

How this relates to what I have/will learn in WDD.

The main benefit of a drop-down menu is the ability to provide the user with expedited access to more specific content. This sense of user focused design re-inforces the concepts discussed in many of the previous design classes. While I will initially develop this statically, the eCommerce nature enables me to consider future development. Dynamically generated content from a database and refreshing cart items using Javascript.

How might I build this?

  1. Create the component as a <header>
  2. Company branding established using <img> or <h1>
  3. Navigation <nav> containing a <ul> with four <li>, each containing an <a>
  4. Dropdown menu items will be nested <ul> inside of corresponding <li>
  5. eCommerce cart view perhaps a <table> inside of last <li>

Responsive Features List

What I hope to practice/learn?

Playing around with media queries has enabled me to see how responsive development works. It seems like every website has to be responsive nowadays. I'd like to continue practicing this by working with additional data. This should allow me to get multiple breakpoints within a 'mobile-first' layout.

How this relates to what I have/will learn in WDD.

The concept of 'mobile-first' has been previously discussed in class. From a design point-of-view it allows us to focus on the most important content.
